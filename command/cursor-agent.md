cursor-agent
===

Cursor IDE 中的 AI 编程助手

## 补充说明

**cursor-agent** 是 Cursor IDE 内置的 AI 编程助手，基于大型语言模型，可以帮助开发者进行代码编写、调试、重构、解释和优化。它能够理解上下文，提供智能代码补全、错误修复建议和代码解释等功能。

###  语法

```shell
# 在 Cursor IDE 中使用
Cmd/Ctrl + K          # 内联编辑模式
Cmd/Ctrl + L          # 打开聊天面板
Cmd/Ctrl + I          # 组合编辑模式
```

###  选项

```shell
# 快捷键操作
Cmd/Ctrl + K          # 内联编辑：在代码中直接编辑，AI 会根据上下文修改代码
Cmd/Ctrl + L          # 聊天模式：打开侧边栏聊天窗口，可以与 AI 对话
Cmd/Ctrl + I          # 组合编辑：在编辑器中打开组合编辑面板

# 聊天命令
@文件名                # 引用特定文件到上下文中
@文件夹名              # 引用整个文件夹到上下文中
@代码库                # 引用整个代码库到上下文中
@最近的文件            # 引用最近打开的文件
@网络搜索              # 启用网络搜索功能
@文档                  # 引用项目文档

# 代码操作
Ctrl + Enter          # 接受 AI 建议
Esc                   # 取消当前操作
Tab                   # 接受部分建议
```

###  参数

**内联编辑模式**：在代码中选中文本后，按 `Cmd/Ctrl + K`，输入指令进行编辑。

**聊天模式**：按 `Cmd/Ctrl + L` 打开聊天面板，可以：
- 询问代码相关问题
- 请求代码生成
- 获取代码解释
- 进行代码重构建议

**组合编辑模式**：按 `Cmd/Ctrl + I` 打开组合编辑面板，可以：
- 同时编辑多个位置
- 进行大规模重构
- 生成多个相关文件

###  实例

#### 基本代码生成

在聊天面板中请求生成代码：

```
请帮我创建一个 React 组件，用于显示用户列表
```

#### 代码解释

选中代码后，在聊天面板中询问：

```
这段代码的作用是什么？
```

#### 代码重构

选中代码后，按 `Cmd/Ctrl + K`，输入：

```
重构这段代码，使其更易读
```

#### 错误修复

当代码有错误时，AI 会自动检测并建议修复：

```
修复这个 TypeScript 类型错误
```

#### 代码补全

在编写代码时，AI 会自动提供智能补全建议：

```typescript
// 输入：const user = 
// AI 会自动补全：const user = { name: string, age: number }
```

#### 使用文件引用

在聊天中使用 `@` 符号引用文件：

```
@utils.ts 请解释这个工具文件中的函数
```

#### 多文件编辑

使用组合编辑模式同时修改多个文件：

```
创建一个用户认证系统，包括登录、注册和密码重置功能
```

#### 代码审查

请求 AI 审查代码：

```
请审查这段代码，找出潜在的问题和改进建议
```

#### 生成测试代码

请求生成测试用例：

```
为这个函数生成单元测试
```

#### 文档生成

请求生成代码文档：

```
为这个类生成 JSDoc 注释
```

#### 代码翻译

将代码从一种语言转换为另一种：

```
将这个 Python 函数转换为 JavaScript
```

#### 性能优化

请求性能优化建议：

```
优化这个函数的性能
```

#### 使用网络搜索

启用网络搜索获取最新信息：

```
@网络搜索 最新的 React 18 特性有哪些？
```

## 扩展知识

### 最佳实践

1. **提供清晰的上下文**：在提问时，尽量提供足够的上下文信息，包括：
   - 项目类型和框架
   - 相关文件引用
   - 具体需求描述

2. **使用文件引用**：使用 `@文件名` 引用相关文件，让 AI 更好地理解代码结构。

3. **分步骤进行**：对于复杂任务，将其分解为多个小步骤，逐步完成。

4. **审查 AI 建议**：始终审查 AI 生成的代码，确保符合项目规范和最佳实践。

5. **利用组合编辑**：对于涉及多个文件的更改，使用组合编辑模式更高效。

### 常用提示词模板

**代码生成**：
```
创建一个 [组件/函数/类]，功能是 [具体功能描述]，使用 [技术栈/框架]
```

**代码重构**：
```
重构这段代码，使其 [更易读/更高效/更符合最佳实践]
```

**错误修复**：
```
修复这个错误：[错误描述或错误信息]
```

**代码解释**：
```
解释这段代码的工作原理：[选中代码]
```

**代码优化**：
```
优化这段代码的 [性能/可读性/可维护性]
```

### 注意事项

1. **隐私和安全**：注意不要在代码中包含敏感信息（如 API 密钥、密码等）。

2. **代码审查**：AI 生成的代码需要人工审查，确保正确性和安全性。

3. **版本兼容性**：AI 可能不了解项目特定的版本要求，需要手动调整。

4. **性能考虑**：AI 建议的代码可能不是最优解，需要根据实际情况优化。

5. **学习工具**：将 AI 助手作为学习和辅助工具，而不是完全依赖它。
